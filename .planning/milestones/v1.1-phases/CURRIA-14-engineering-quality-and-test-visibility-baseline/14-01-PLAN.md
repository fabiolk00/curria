# Plan 14-01

## Goal

Introduce a TypeScript-aware linting baseline that fits the brownfield repo.

## Tasks

1. Audit the current ESLint setup and identify the minimal `typescript-eslint` rules that add value without exploding noise.
2. Stage enforcement so the team can apply stricter rules to touched files or selected directories first.
3. Add scripts and docs for running the new lint contract locally and in CI.
4. Prove the rollout does not create unreasonable churn for unrelated legacy code.

## Verification

- Lint command passes with the chosen staged scope
- Documentation explains the rollout and enforcement boundaries
- Representative touched-file scenarios are covered
