# Plan 14-03

## Goal

Make the testing story visible and actionable for contributors.

## Tasks

1. Document the existing test layers:
   - unit
   - integration
   - route
   - browser E2E
   - stress or operator checks where relevant
2. Map the existing commands and representative suites to those layers.
3. Define coverage expectations for new work so contributors know what to run and what to add.
4. Keep the docs grounded in actual scripts and maintained suites.

## Verification

- Committed testing guide reflects real commands and suites
- Contributor-facing docs explain coverage expectations clearly
- At least one phase artifact or checklist references the documented testing contract
