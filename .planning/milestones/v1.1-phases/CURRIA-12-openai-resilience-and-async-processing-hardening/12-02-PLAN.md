# Plan 12-02

## Goal

Remove the highest-risk synchronous PDF processing path from route-timeout exposure.

## Tasks

1. Map the current PDF upload and parsing request lifecycle, including payload size and provider-cost seams.
2. Introduce staged or deferred processing for heavy PDF work:
   - create pending state
   - process asynchronously or in resumable stages
   - expose completion and failure states back to the UI
3. Keep small or cheap flows working while heavier documents move through the new staged path.
4. Preserve factual extraction behavior and existing brownfield UX where possible.

## Verification

- Tests for pending, completed, and failed PDF-processing states
- Stress coverage with larger PDF fixtures
- Route-level checks proving uploads do not depend on one long synchronous parse window
