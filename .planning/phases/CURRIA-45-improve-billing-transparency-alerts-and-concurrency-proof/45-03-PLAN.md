## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| concurrent export requests -> billing state machine | Duplicate or racing requests can cross the financial mutation boundary |
| staging scripts -> authenticated environment | Stress and snapshot helpers need credentials but must stay read-only except for the explicit test flow |
| anomaly summary -> operator action | False positives or missing thresholds can either hide or overstate financial risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-45-10 | R | staging diagnostics | mitigate | Capture reservation, ledger, and anomaly evidence in scripts and docs so operators can attribute every alert to concrete data. |
| T-45-11 | I | staging helper output | mitigate | Keep sensitive snapshots scoped to existing authenticated env vars and do not expose helper routes publicly. |

</threat_model>
