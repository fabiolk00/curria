<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| export orchestration -> storage renderer | Untrusted runtime and storage failures can occur after validation succeeds |
| export orchestration -> billing RPCs | Credit mutation crosses into billing-sensitive persistence and may drift by environment |
| export orchestration -> resume generation persistence | Auxiliary history writes can fail independently of artifact success |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | T | `generate-billable-resume.ts` | mitigate | Keep artifact, billing, and persistence stages explicit so one late write cannot tamper with the already-successful artifact outcome. |
| T-43-02 | D | `consumeCreditForGeneration` | mitigate | Fall back only to the existing generic atomic credit path when generation-specific billing infrastructure is missing or drifted; do not silently skip consumption. |
| T-43-03 | R | export/billing logs | mitigate | Add structured warning logs that record whether failure happened during render, billing, or persistence so operators can diagnose the exact stage. |
| T-43-04 | I | generation failure responses | accept | Preserve current user-facing error wording unless correctness requires a change; this phase is operational hardening, not UX redesign. |
</threat_model>
