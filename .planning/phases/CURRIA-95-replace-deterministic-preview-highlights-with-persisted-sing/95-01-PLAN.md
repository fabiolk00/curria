<objective>
Replace the deterministic preview highlight architecture with one persisted LLM-generated highlight artifact that is produced once from the finalized optimized resume and rendered without mutating `cvState`.

Purpose: eliminate client-side heuristic recomputation, remove the legacy metric-preservation preview subsystem, and keep highlight lifecycle aligned with optimized resume persistence, rollback, and preview-lock rules.
Output: shared highlight artifact contracts/helpers, a single-call detector, pipeline persistence/rollback plumbing, thin route serialization, and a renderer migrated off `optimized-preview-highlights.ts`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OpenAI detector -> persisted session state | Untrusted model output crosses into stored highlight artifacts. |
| server comparison/session routes -> browser | Highlight metadata can reveal optimized content if preview-lock rules drift. |
| optimized resume lifecycle -> manual edit/reset flows | Stale highlight artifacts can survive rollback or optimized-state replacement. |

| T-95-02 | I | `session-comparison/decision.ts`, `app/api/session/[id]/route.ts` | mitigate | Omit `highlightState` whenever `sanitizeGeneratedCvStateForClient(...)` hides optimized content, and cover locked-preview responses in route tests. |
| T-95-03 | T | ATS/job-targeting pipelines, manual-edit route, request orchestrator | mitigate | Snapshot/restore `previousHighlightState` with `optimizedCvState`, and clear `highlightState` in every reset/edit seam that replaces optimized content. |
